### **Системные инструкции для обработки кода проекта `hypotez`**

=========================================================================================

Описание функциональности и правил для генерации, анализа и улучшения кода. Направлено на обеспечение последовательного и читаемого стиля кодирования, соответствующего требованиям.

---

### **Основные принципы**

#### **1. Общие указания**:
- Соблюдай четкий и понятный стиль кодирования.
- Все изменения должны быть обоснованы и соответствовать установленным требованиям.

#### **2. Комментарии**:
- Используй `#` для внутренних комментариев.
- Документация всех функций, методов и классов должна следовать такому формату: 
    ```python
        def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
            """ 
            Args:
                param (str): Описание параметра `param`.
                param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.
    
            Returns:
                dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.
    
            Raises:
                SomeError: Описание ситуации, в которой возникает исключение `SomeError`.

            Ехаmple:
                >>> function('param', 'param1')
                {'param': 'param1'}
            """
    ```
- Комментарии и документация должны быть четкими, лаконичными и точными.

#### **3. Форматирование кода**:
- Используй одинарные кавычки. `a:str = 'value'`, `print('Hello World!')`;
- Добавляй пробелы вокруг операторов. Например, `x = 5`;
- Все параметры должны быть аннотированы типами. `def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:`;
- Не используй `Union`. Вместо этого используй `|`.

#### **4. Логирование**:
- Для логгирования Всегда Используй модуль `logger` из `src.logger.logger`.
- Ошибки должны логироваться с использованием `logger.error`.
Пример:
    ```python
        try:
            ...
        except Exception as ex:
            logger.error('Error while processing data', ех, exc_info=True)
    ```
#### **5 Не используй `Union[]` в коде. Вместо него используй `|`
Например:
```python
x: str | int ...
```

### Анализ кода модуля `src.utils.convertors.tts.py`

#### 1. Блок-схема:

```mermaid
graph LR
    A[Начало: speech_recognizer] --> B{Есть audio_url?}
    B -- Да --> C[Скачать аудиофайл по URL]
    C --> D[Сохранить аудиофайл во временную директорию]
    B -- Нет --> E{Есть audio_file_path?}
    E -- Да --> F[Использовать существующий файл]
    E -- Нет --> Z[Выдать ошибку, нет файла для обработки]
    D --> G[Конвертировать OGG в WAV]
    F --> G
    G --> H[Инициализировать recognizer]
    H --> I[Открыть WAV файл как аудиоисточник]
    I --> J[Записать аудиоданные из источника]
    J --> K{Попытка распознавания речи через Google Speech Recognition}
    K -- Успешно --> L[Вернуть распознанный текст]
    K -- sr.UnknownValueError --> M[Логировать ошибку: "Google Speech Recognition could not understand audio"]
    M --> N[Вернуть сообщение об ошибке]
    K -- sr.RequestError --> O[Логировать ошибку: "Could not request results from Google Speech Recognition service"]
    O --> P[Вернуть сообщение об ошибке]
    L --> Q[Конец: speech_recognizer (успех)]
    N --> Q
    P --> Q
    Z --> Q

    A2[Начало: text2speech] --> B2[Сгенерировать речь из текста, используя gTTS]
    B2 --> C2[Сохранить аудиофайл во временную директорию как MP3]
    C2 --> D2[Конвертировать MP3 в WAV, используя pydub]
    D2 --> E2[Логировать путь к сохраненному WAV файлу]
    E2 --> F2[Вернуть путь к WAV файлу]
    F2 --> G2[Конец: text2speech (успех)]
```

**Примеры для каждого блока:**

- **A**: `speech_recognizer(audio_url="https://example.com/audio.ogg")`
- **B**: Проверка, был ли передан `audio_url`.
- **C**: Скачивание файла по URL с использованием `requests.get()`.
- **D**: Сохранение скачанного файла во временную директорию с использованием `tempfile.gettempdir()`.
- **E**: Проверка, был ли передан `audio_file_path`.
- **F**: Использование существующего локального файла.
- **G**: Преобразование формата OGG в WAV с использованием `pydub.AudioSegment`.
- **H**: Инициализация распознавателя речи `sr.Recognizer()`.
- **I**: Открытие WAV-файла для чтения аудиоданных.
- **J**: Запись аудиоданных из файла с использованием `recognizer.record(source)`.
- **K**: Попытка распознавания речи с использованием `recognizer.recognize_google()`.
- **L**: Возврат распознанного текста.
- **M**: Обработка ошибки, если Google Speech Recognition не смог распознать аудио.
- **N**: Возврат сообщения об ошибке.
- **O**: Обработка ошибки, если не удалось запросить результаты от сервиса Google Speech Recognition.
- **P**: Возврат сообщения об ошибке.
- **Q**: Конец выполнения функции `speech_recognizer`.

- **A2**: `text2speech(text="Привет")`
- **B2**: Генерация речи из текста с использованием `gTTS`.
- **C2**: Сохранение сгенерированного аудиофайла во временную директорию в формате MP3.
- **D2**: Конвертация MP3-файла в WAV-файл с использованием `pydub.AudioSegment`.
- **E2**: Логирование пути к сохраненному WAV-файлу.
- **F2**: Возврат пути к WAV-файлу.
- **G2**: Конец выполнения функции `text2speech`.

**Объяснение зависимостей:**

- `speech_recognition`: Используется для распознавания речи в аудиофайле.
- `pydub`: Применяется для конвертации аудиоформатов, таких как OGG в WAV и MP3 в WAV.
- `gtts`: Используется для преобразования текста в речь.
- `requests`: Применяется для скачивания аудиофайлов по URL.
- `src.utils.jjson`: Используется для загрузки JSON конфигураций, например, настроек для gTTS или speech_recognition.
- `src.logger.logger`: Используется для логирования ошибок и информации.
- `pathlib`: Используется для работы с путями к файлам.
- `tempfile`: Используется для создания временных файлов для хранения скачанных и преобразованных аудиофайлов.
- `asyncio`: Используется для асинхронного выполнения функции `text2speech`.

#### 3. Объяснение:

- **Импорты:**
  - `pathlib`: Используется для работы с путями к файлам и директориям в объектно-ориентированном стиле. Это упрощает манипуляции с файлами и директориями.
  - `tempfile`: Предоставляет функции для создания временных файлов и директорий. В данном коде используется для хранения скачанных и преобразованных аудиофайлов.
  - `asyncio`: Используется для асинхронного выполнения функции `text2speech`, что позволяет не блокировать основной поток выполнения.
  - `requests`: Используется для отправки HTTP-запросов, например, для скачивания аудиофайла по URL.
  - `speech_recognition as sr`: Библиотека для распознавания речи. Позволяет преобразовывать аудио в текст.
  - `pydub`: Используется для работы с аудиофайлами, включая конвертацию форматов.
  - `gtts`: Библиотека Google Text-to-Speech для преобразования текста в речь.
  - `src.utils.jjson`: Модуль, содержащий функции `j_loads` и `j_loads_ns` для загрузки JSON-файлов.
  - `src.logger.logger`: Модуль для логирования событий и ошибок.

- **Функции:**
  - `speech_recognizer(audio_url: str = None, audio_file_path: Path = None, language: str = 'ru-RU') -> str`:
    - **Аргументы**:
      - `audio_url (str, optional)`: URL аудиофайла для скачивания.
      - `audio_file_path (Path, optional)`: Локальный путь к аудиофайлу.
      - `language (str)`: Язык для распознавания речи (например, 'ru-RU').
    - **Возвращаемое значение**:
      - `str`: Распознанный текст из аудио или сообщение об ошибке.
    - **Назначение**:
      - Функция скачивает аудиофайл (если предоставлен `audio_url`) или использует локальный файл (`audio_file_path`), конвертирует его в формат WAV, и затем использует Google Speech Recognition для преобразования аудио в текст. Логирует распознанный текст или ошибки.
    - **Пример**:
      ```python
      recognized_text = speech_recognizer(audio_url='https://example.com/audio.ogg')
      print(recognized_text)  # Output: "Привет"
      ```

  - `text2speech(text: str, lang: str = 'ru') -> str`:
    - **Аргументы**:
      - `text (str)`: Текст для преобразования в речь.
      - `lang (str)`: Язык для озвучивания (например, 'ru').
    - **Возвращаемое значение**:
      - `str`: Путь к сгенерированному аудиофайлу.
    - **Назначение**:
      - Функция использует Google Text-to-Speech (`gtts`) для преобразования текста в речь, сохраняет его как MP3 файл, затем конвертирует MP3 в WAV с использованием `pydub`. Логирует путь к сохраненному WAV файлу.
    - **Пример**:
      ```python
      audio_path = await text2speech('Привет', lang='ru')
      print(audio_path)  # Output: "/tmp/response.wav"
      ```

- **Переменные**:
  - `audio_url (str)`: URL для скачивания аудиофайла.
  - `audio_file_path (Path)`: Путь к локальному аудиофайлу.
  - `language (str)`: Язык для распознавания речи.
  - `text (str)`: Текст для преобразования в речь.
  - `lang (str)`: Язык для озвучивания.
  - `recognizer (sr.Recognizer)`: Объект распознавателя речи из библиотеки `speech_recognition`.
  - `audio (AudioSegment)`: Объект аудио сегмента из библиотеки `pydub`.
  - `tts (gTTS)`: Объект преобразования текста в речь из библиотеки `gtts`.

- **Потенциальные ошибки и области для улучшения**:
  - Обработка исключений: В функциях `speech_recognizer` и `text2speech` предусмотрена общая обработка исключений, но можно добавить более специфичные блоки `except` для разных типов ошибок, чтобы более точно реагировать на них.
  - Временные файлы: Код создает временные файлы, но не всегда удаляет их после использования. Можно добавить удаление временных файлов после завершения работы функций, чтобы избежать накопления мусора.
  - Зависимости: Для работы кода требуются установленные библиотеки `speech_recognition`, `pydub`, `gtts` и `requests`. Необходимо убедиться, что эти зависимости установлены в окружении выполнения.

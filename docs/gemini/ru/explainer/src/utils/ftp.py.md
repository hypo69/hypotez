### **Системные инструкции для обработки кода проекта `hypotez`**

=========================================================================================

Описание функциональности и правил для генерации, анализа и улучшения кода. Направлено на обеспечение последовательного и читаемого стиля кодирования, соответствующего требованиям.

---

### **Основные принципы**

#### **1. Общие указания**:
- Соблюдай четкий и понятный стиль кодирования.
- Все изменения должны быть обоснованы и соответствовать установленным требованиям.

#### **2. Комментарии**:
- Используй `#` для внутренних комментариев.
- Документация всех функций, методов и классов должна следовать такому формату: 
    ```python
        def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
            """ 
            Args:
                param (str): Описание параметра `param`.
                param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.
    
            Returns:
                dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.
    
            Raises:
                SomeError: Описание ситуации, в которой возникает исключение `SomeError`.

            Ехаmple:
                >>> function('param', 'param1')
                {'param': 'param1'}
            """
    ```
- Комментарии и документация должны быть четкими, лаконичными и точными.

#### **3. Форматирование кода**:
- Используй одинарные кавычки. `a:str = 'value'`, `print('Hello World!')`;
- Добавляй пробелы вокруг операторов. Например, `x = 5`;
- Все параметры должны быть аннотированы типами. `def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:`;
- Не используй `Union`. Вместо этого используй `|`.

#### **4. Логирование**:
- Для логгирования Всегда Используй модуль `logger` из `src.logger.logger`.
- Ошибки должны логироваться с использованием `logger.error`.
Пример:
    ```python
        try:
            ...
        except Exception as ex:
            logger.error('Error while processing data', ех, exc_info=True)
    ```
#### **5 Не используй `Union[]` в коде. Вместо него используй `|`
Например:
```python
x: str | int ...
```

- Вместо: *«получаем»*, *«возвращаем»*, *«преобразовываем»* используй имя объекта *«функция получае»*, *«переменная возвращает»*, *«код преобразовывает»*

#### **5. Сохранение комментариев**:

- Если комментарий кажется устаревшим или неясным, не изменяйте его. Вместо этого отметьте его в разделе «Изменения».

#### **6. Обработка `...` в коде**:
- Оставляйте `...` как указатели в коде без изменений.
- Не документируйте строки с `...`.

### Анализ кода `hypotez/src/utils/ftp.py`

#### 1. Блок-схема

```mermaid
graph TD
    A[Начало операции] --> B{Установить FTP соединение};
    B -- Успешно --> C{Выбрать директорию на FTP};
    B -- Ошибка --> E[Логировать ошибку соединения];
    C --> D{Выполнить операцию с файлом};
    D -- Успешно --> F{Закрыть FTP соединение};
    D -- Ошибка --> G[Логировать ошибку операции];
    F --> H[Завершение операции];
    G --> F;
    E --> H;
    
    subgraph write(source_file_path, dest_dir, dest_file_name)
        D -- Запись --> D1[Открыть файл для чтения];
        D1 --> D2{Передать файл на FTP};
        D2 -- Успешно --> D;
        D2 -- Ошибка --> G;
    end
    
    subgraph read(source_file_path, dest_dir, dest_file_name)
        D -- Чтение --> D3[Открыть файл для записи];
        D3 --> D4{Получить файл с FTP};
        D4 --> D5[Открыть файл для чтения];
        D5 --> D;
        D4 -- Ошибка --> G;
    end
    
    subgraph delete(source_file_path, dest_dir, dest_file_name)
        D -- Удаление --> D6{Удалить файл на FTP};
        D6 -- Успешно --> D;
        D6 -- Ошибка --> G;
    end
```

**Примеры для логических блоков:**

*   **A (Начало операции)**: Начало вызова функции `write`, `read` или `delete`.
*   **B (Установить FTP соединение)**: `session = ftplib.FTP(_connection['server'], _connection['user'], _connection['password'])`. Если соединение не удалось, генерируется исключение, которое перехватывается и логируется.
*   **C (Выбрать директорию на FTP)**: `session.cwd(dest_dir)`.
*   **D (Выполнить операцию с файлом)**: В зависимости от операции (чтение, запись, удаление) вызываются соответствующие методы `ftplib`:
    *   Запись: `session.storbinary(f'STOR {dest_file_name}', f)`
    *   Чтение: `session.retrbinary(f'RETR {dest_file_name}', f.write)`
    *   Удаление: `session.delete(dest_file_name)`
*   **E (Логировать ошибку соединения)**: `logger.error(f"Failed to connect to FTP server. Error: {ex}")`.
*   **F (Закрыть FTP соединение)**: `session.quit()`.
*   **G (Логировать ошибку операции)**: `logger.error(f"Failed to send file to FTP server. Error: {ex}")`.
*   **H (Завершение операции)**: Завершение вызова функции `write`, `read` или `delete`.

**Анализ зависимостей:**

*   `ftplib`: Используется для установления FTP-соединения, навигации по директориям и выполнения операций с файлами (запись, чтение, удаление).
*   `_connection`: Словарь, содержащий параметры соединения с FTP-сервером (сервер, пользователь, пароль).
*   `logger`: Используется для логирования ошибок и исключений.

#### 3. Объяснение

*   **Импорты:**
    *   `from src.logger.logger import logger`: Импортирует модуль `logger` для логирования операций и ошибок. Логгер используется для записи информации о процессе выполнения функций, что полезно для отладки и мониторинга.
    *   `from typing import Union`: Импортирует `Union` для обозначения, что функция может возвращать разные типы данных.
    *   `import ftplib`: Импортирует модуль `ftplib`, который предоставляет функциональность для взаимодействия с FTP-серверами. Этот модуль позволяет устанавливать соединения, отправлять, получать и удалять файлы.
    *   `from pathlib import Path`: Импортирует класс `Path` из модуля `pathlib` для работы с путями к файлам в файловой системе. В данном коде не используется, но может быть полезен для расширения функциональности.
*   **Переменные:**
    *   `_connection`: Словарь, содержащий параметры для подключения к FTP-серверу, такие как адрес сервера, порт, имя пользователя и пароль. Эти параметры используются для установления соединения с FTP-сервером в функциях `write`, `read` и `delete`.
*   **Функции:**
    *   `write(source_file_path: str, dest_dir: str, dest_file_name: str) -> bool`:
        *   **Аргументы:**
            *   `source_file_path` (str): Путь к файлу, который нужно отправить на FTP-сервер.
            *   `dest_dir` (str): Директория на FTP-сервере, куда нужно поместить файл.
            *   `dest_file_name` (str): Имя файла на FTP-сервере.
        *   **Возвращаемое значение:**
            *   `bool`: `True`, если файл успешно отправлен, `False` в противном случае.
        *   **Назначение:**
            *   Отправляет файл на FTP-сервер. Устанавливает соединение, переходит в указанную директорию и отправляет файл.
        *   **Пример:**
            ```python
            success = write('local_path/to/file.txt', '/remote/directory', 'file.txt')
            if success:
                print("Файл успешно отправлен")
            else:
                print("Ошибка при отправке файла")
            ```
    *   `read(source_file_path: str, dest_dir: str, dest_file_name: str) -> Union[str, bytes, None]`:
        *   **Аргументы:**
            *   `source_file_path` (str): Путь, куда нужно сохранить полученный с FTP-сервера файл.
            *   `dest_dir` (str): Директория на FTP-сервере, где находится файл.
            *   `dest_file_name` (str): Имя файла на FTP-сервере.
        *   **Возвращаемое значение:**
            *   `Union[str, bytes, None]`: Содержимое файла, если файл успешно получен, `None` в противном случае.
        *   **Назначение:**
            *   Получает файл с FTP-сервера. Устанавливает соединение, переходит в указанную директорию и получает файл.
        *   **Пример:**
            ```python
            content = read('local_path/to/file.txt', '/remote/directory', 'file.txt')
            if content:
                print(f"Содержимое файла: {content[:100]}...")
            else:
                print("Ошибка при получении файла")
            ```
    *   `delete(source_file_path: str, dest_dir: str, dest_file_name: str) -> bool`:
        *   **Аргументы:**
            *   `source_file_path` (str): Путь, где файл находится локально (не используется в функции).
            *   `dest_dir` (str): Директория на FTP-сервере, где находится файл.
            *   `dest_file_name` (str): Имя файла на FTP-сервере.
        *   **Возвращаемое значение:**
            *   `bool`: `True`, если файл успешно удален, `False` в противном случае.
        *   **Назначение:**
            *   Удаляет файл с FTP-сервера. Устанавливает соединение, переходит в указанную директорию и удаляет файл.
        *   **Пример:**
            ```python
            success = delete('local_path/to/file.txt', '/remote/directory', 'file.txt')
            if success:
                print("Файл успешно удален")
            else:
                print("Ошибка при удалении файла")
            ```
*   **Потенциальные ошибки и области для улучшения:**
    *   **Обработка исключений:** В блоках `try...except...finally` происходит логирование ошибок, но не всегда осуществляется их дальнейшая обработка. Возможно, стоит добавить повторные попытки выполнения операций или более детальную обработку ошибок в зависимости от их типа.
    *   **Безопасность:** Хранение учетных данных в коде (в словаре `_connection`) небезопасно. Рекомендуется использовать переменные окружения или другие способы хранения секретов.
    *   **Отсутствие обработки `source_file_path` в `delete`:** Аргумент `source_file_path` не используется в функции `delete`. Это может быть упущением или оставшимся артефактом от предыдущей реализации.
    *   **Типы возвращаемых значений:** Функция `read` возвращает `Union[str, bytes, None]`. Это может быть неудобно, так как вызывающей стороне нужно проверять тип возвращаемого значения. Лучше определить один тип возвращаемого значения (например, `bytes`), а в случае ошибки возвращать пустую строку или `None`.
    *   **Отсутствие обработки кодировки:** При чтении файла не указана кодировка, что может привести к проблемам при работе с файлами, содержащими символы, отличные от ASCII.

**Цепочка взаимосвязей с другими частями проекта:**

*   `src.logger.logger`: Этот модуль используется для логирования ошибок и событий, происходящих в функциях `write`, `read` и `delete`. Логирование помогает отслеживать и отлаживать операции с FTP-сервером.
*   Параметры соединения с FTP-сервером (`_connection`) могут быть получены из конфигурационных файлов или переменных окружения, что подразумевает связь с модулями, отвечающими за конфигурацию приложения.

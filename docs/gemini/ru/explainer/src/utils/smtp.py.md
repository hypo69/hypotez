### **Системные инструкции для обработки кода проекта `hypotez`**

=========================================================================================

Описание функциональности и правил для генерации, анализа и улучшения кода. Направлено на обеспечение последовательного и читаемого стиля кодирования, соответствующего требованиям.

---

### **Основные принципы**

#### **1. Общие указания**:
- Соблюдай четкий и понятный стиль кодирования.
- Все изменения должны быть обоснованы и соответствовать установленным требованиям.

#### **2. Комментарии**:
- Используй `#` для внутренних комментариев.
- Документация всех функций, методов и классов должна следовать такому формату: 
    ```python
        def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
            """ 
            Args:
                param (str): Описание параметра `param`.
                param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.
    
            Returns:
                dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.
    
            Raises:
                SomeError: Описание ситуации, в которой возникает исключение `SomeError`.

            Ехаmple:
                >>> function('param', 'param1')
                {'param': 'param1'}
            """
    ```
- Комментарии и документация должны быть четкими, лаконичными и точными.

#### **3. Форматирование кода**:
- Используй одинарные кавычки. `a:str = 'value'`, `print('Hello World!')`;
- Добавляй пробелы вокруг операторов. Например, `x = 5`;
- Все параметры должны быть аннотированы типами. `def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:`;
- Не используй `Union`. Вместо этого используй `|`.

#### **4. Логирование**:
- Для логгирования Всегда Используй модуль `logger` из `src.logger.logger`.
- Ошибки должны логироваться с использованием `logger.error`.
Пример:
    ```python
        try:
            ...
        except Exception as ex:
            logger.error('Error while processing data', ех, exc_info=True)
    ```
#### **5 Не используй `Union[]` в коде. Вместо него используй `|`
Например:
```python
x: str | int ...
```

### Анализ кода `hypotez/src/utils/smtp.py`

#### 1. Блок-схема:

```mermaid
graph LR
    A[Start] --> B{Получение параметров SMTP/IMAP из окружения};
    B -- Да --> C{Функция send(subject, body, to)};
    B -- Нет --> E[Логирование ошибки, завершение];
    C --> D{Создание SMTP соединения};
    D --> F{Настройка TLS};
    F --> G{Авторизация на SMTP сервере};
    G --> H{Создание MIME сообщения};
    H --> I{Отправка сообщения};
    I --> J{Завершение SMTP соединения};
    J --> K[Успех: return True];
    C -- Ошибка --> L[Логирование ошибки, return False];
    A --> M{Функция receive(imap_server, user, password, folder)};
    M --> N{Подключение к IMAP серверу};
    N --> O{Авторизация на IMAP сервере};
    O --> P{Выбор почтовой папки};
    P --> Q{Поиск всех email};
    Q --> R{Получение email по ID};
    R --> S{Извлечение данных (subject, from, body)};
    S --> T{Добавление в список};
    T --> U{Закрытие IMAP соединения};
    U --> V{Выход из IMAP};
    V --> W[Успех: return List[Dict]];
    M -- Ошибка --> X[Логирование ошибки, return None];
```

**Примеры для логических блоков:**

*   **B**: Функция `os.environ.get('SMTP_SERVER', 'smtp.example.com')` пытается получить значение переменной окружения `SMTP_SERVER`. Если она не установлена, используется значение по умолчанию `'smtp.example.com'`.
*   **D**: `smtplib.SMTP(_connection['server'], _connection['port'])` создает SMTP соединение, используя сервер и порт, полученные из конфигурации.
*   **G**: `smtp.login(_connection['user'], _connection['password'])` выполняет вход на SMTP сервер, используя имя пользователя и пароль.
*   **I**: `smtp.sendmail(_connection['user'], to, message.as_string())` отправляет email, используя указанного отправителя, получателя и тело сообщения.
*   **N**: `imaplib.IMAP4_SSL(imap_server)` устанавливает защищенное SSL-соединение с IMAP-сервером.
*   **O**: `mail.login(user, password)` выполняет вход в почтовый ящик IMAP-сервера.
*   **S**: `msg.get_payload(decode=True, _charset="utf-8").decode("utf-8", "ignore")` извлекает тело сообщения из email, декодирует его и обрабатывает возможные ошибки кодировки.

**Объяснение зависимостей:**

*   `smtplib`: Используется для отправки email через SMTP. Функция `send` использует `smtplib.SMTP` для установления соединения с SMTP сервером, авторизации и отправки email сообщений.
*   `imaplib`: Используется для получения email через IMAP. Функция `receive` использует `imaplib.IMAP4_SSL` для установления защищенного соединения с IMAP сервером, авторизации и получения email сообщений.
*   `email`: Используется для создания и разбора email сообщений. Функция `send` использует `email.mime.text.MIMEText` для создания email сообщения. Функция `receive` использует `email.message_from_bytes` для разбора полученного email сообщения.
*   `os`: Используется для получения параметров конфигурации из переменных окружения. Функции `send` и `receive` используют `os.environ.get` для получения параметров SMTP и IMAP сервера, имени пользователя и пароля.
*   `typing`: Используется для аннотации типов.  Определяет типы данных для аргументов и возвращаемых значений функций `send` и `receive`.
*   `src.logger.logger`: Используется для логирования ошибок.  Функции `send` и `receive` используют `logger.error` для записи информации об ошибках в лог.

#### 3. Объяснение:

**Импорты:**

*   `smtplib`: Предоставляет классы и функции для отправки email сообщений с использованием протокола SMTP. Используется функцией `send` для отправки сообщений.
*   `imaplib`: Предоставляет классы и функции для получения email сообщений с использованием протокола IMAP. Используется функцией `receive` для получения сообщений.
*   `email`: Предоставляет классы и функции для работы с email сообщениями, включая создание, разбор и форматирование. Используется для создания MIME сообщения в функции `send` и для разбора полученных сообщений в функции `receive`.
*   `os`: Предоставляет функции для взаимодействия с операционной системой, включая доступ к переменным окружения. Используется для получения параметров конфигурации (сервер, порт, пользователь, пароль) из переменных окружения.
*   `email.mime.text.MIMEText`: Используется для создания email сообщения в формате MIME text. Позволяет задать тело сообщения, тему и кодировку.
*   `typing.List`, `typing.Dict`, `typing.Optional`: Используются для аннотации типов данных. Указывают типы аргументов и возвращаемых значений функций, что улучшает читаемость и упрощает отладку.
*   `src.logger.logger`: Пользовательский модуль логирования, используемый для записи ошибок и другой полезной информации о работе функций.

**Переменные:**

*   `_connection`: Словарь, содержащий параметры подключения к SMTP серверу.  **Важно:**  Значения (сервер, порт, пользователь, пароль, получатель) должны быть получены из переменных окружения, а **не** закодированы непосредственно в коде. Это необходимо для обеспечения безопасности.

**Функции:**

*   `send(subject: str = '', body: str = '', to: str = 'one.last.bit@gmail.com') -> bool`:
    *   Аргументы:
        *   `subject` (str): Тема email сообщения. По умолчанию пустая строка.
        *   `body` (str): Тело email сообщения. По умолчанию пустая строка.
        *   `to` (str): Адрес получателя email сообщения. По умолчанию `'one.last.bit@gmail.com'`.
    *   Возвращаемое значение:
        *   `bool`: `True` в случае успешной отправки, `False` в случае ошибки.
    *   Назначение:
        *   Отправляет email сообщение, используя параметры подключения из словаря `_connection`.
        *   Создает SMTP соединение, выполняет аутентификацию, создает MIME сообщение и отправляет его.
        *   В случае ошибки логирует информацию об ошибке с использованием модуля `logger`.
    *   Пример:
        ```python
        if send(subject='Тест', body='Это тестовое сообщение', to='test@example.com'):
            print('Сообщение успешно отправлено')
        else:
            print('Ошибка при отправке сообщения')
        ```
*   `receive(imap_server: str, user: str, password: str, folder: str = 'inbox') -> Optional[List[Dict[str, str]]]`:
    *   Аргументы:
        *   `imap_server` (str): Адрес IMAP сервера.
        *   `user` (str): Имя пользователя для подключения к IMAP серверу.
        *   `password` (str): Пароль для подключения к IMAP серверу.
        *   `folder` (str): Название почтовой папки для получения сообщений. По умолчанию `'inbox'`.
    *   Возвращаемое значение:
        *   `Optional[List[Dict[str, str]]]`: Список словарей, где каждый словарь содержит информацию об email сообщении (тема, отправитель, тело). Возвращает `None` в случае ошибки.
    *   Назначение:
        *   Получает email сообщения из указанной почтовой папки на IMAP сервере.
        *   Устанавливает IMAP соединение, выполняет аутентификацию, выбирает почтовую папку, ищет все сообщения и извлекает информацию о каждом сообщении.
        *   В случае ошибки логирует информацию об ошибке с использованием модуля `logger`.
    *   Пример:
        ```python
        emails = receive(imap_server='imap.example.com', user='test@example.com', password='password')
        if emails:
            for email in emails:
                print(f"Тема: {email['subject']}")
                print(f"Отправитель: {email['from']}")
                print(f"Тело: {email['body'][:100]}...") # Выводим только первые 100 символов тела
        else:
            print('Ошибка при получении сообщений')
        ```

**Потенциальные ошибки и области для улучшения:**

*   **Безопасность:** Критически важно, чтобы параметры подключения (особенно пароли) не хранились непосредственно в коде. Использование переменных окружения - правильный подход, но необходимо убедиться, что переменные окружения настроены корректно и безопасно.
*   **Обработка исключений:** Код включает обработку исключений, что является хорошей практикой.  Однако, можно рассмотреть более гранулярную обработку исключений, чтобы, например, различать ошибки аутентификации от ошибок соединения.
*   **Кодировка:** При получении email сообщений кодировка явно указывается как `"utf-8"`.  Возможно, стоит предусмотреть определение кодировки из заголовков сообщения или использовать более универсальный подход для обработки различных кодировок.
*   **Параметризация**: Можно вынести некоторые константы, такие как порт SMTP по умолчанию или адрес получателя по умолчанию, в отдельные переменные или параметры конфигурации. Это упростит изменение этих значений в будущем.
*   **Повторное использование соединения:**  В текущей реализации функции `send` и `receive` создают новое соединение для каждого вызова. В случае частого использования этих функций, может быть полезно реализовать механизм повторного использования соединения для повышения производительности.
*   **Размер получаемого сообщения:** В функции `receive` извлекаются все сообщения сразу. Если почтовый ящик содержит очень много сообщений, это может привести к проблемам с памятью.  Можно рассмотреть возможность постраничной загрузки сообщений или использования других механизмов для обработки больших объемов данных.

**Взаимосвязь с другими частями проекта:**

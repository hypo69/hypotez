# Управление сайтами на PrestaShop

&nbsp;&nbsp;&nbsp;Документ описывает структуру и способ работы с сайтами на платформе PrestaShop, а также хранение и использование ключей API.

## Сайты

Ваши сайты, работающие на PrestaShop:
1. [e-cat.co.il](https://e-cat.co.il)
2. [emil-design.com](https://emil-design.com)
3. [sergey.mymaster.co.il](https://sergey.mymaster.co.il)

Каждый из этих сайтов использует API для взаимодействия с различными параметрами и функциями.

## Хранение ключей API

Ключи API для каждого сайта хранятся в файле `credentials.kdbx`. Этот файл является защищенной базой данных паролей и содержит следующие данные для каждого сайта:
- URL сайта
- Ключ API
- Дополнительные метаданные (если необходимо)

Для работы с ключами из файла используйте менеджер паролей, поддерживающий формат `.kdbx`, например, [KeePass](https://keepass.info/) или [KeePassXC](https://keepassxc.org/).

### Структура файла `credentials.kdbx`

Файл `credentials.kdbx` представляет собой базу данных, защищенную паролем. Он содержит записи для каждого сайта PrestaShop, включающие:

- **URL сайта**: URL-адрес сайта PrestaShop.
- **Ключ API**: Ключ API для доступа к API PrestaShop.
- **Дополнительные метаданные**: Дополнительная информация, специфичная для каждого сайта (например, описание, контактные данные).

### Установка и настройка KeePassXC

1.  **Скачайте KeePassXC**: Перейдите на [официальный сайт KeePassXC](https://keepassxc.org/download/) и скачайте версию, подходящую для вашей операционной системы.
2.  **Установите KeePassXC**: Следуйте инструкциям по установке для вашей операционной системы.
3.  **Откройте файл `credentials.kdbx`**: Запустите KeePassXC и откройте файл `credentials.kdbx`. Вам потребуется ввести мастер-пароль, который обеспечивает защиту базы данных.

## Пример использования API

Чтобы подключиться к API одного из сайтов, следуйте следующему шаблону:

### Получение API_KEY

1.  **Авторизуйтесь в административной панели PrestaShop**: Войдите в административную панель вашего сайта PrestaShop.
2.  **Перейдите в раздел "Webservice"**: В меню слева найдите и выберите пункт "Webservice" (или "Веб-сервисы").
3.  **Создайте новый ключ API**: Нажмите кнопку "Add new webservice key" (или "Добавить новый ключ веб-сервиса").
4.  **Настройте права доступа**: Установите необходимые права доступа для ключа API (например, чтение, запись, изменение).
5.  **Сохраните ключ API**: Сохраните изменения. Сгенерированный ключ API будет отображен на странице.

### Кодирование API_KEY в Base64

Используйте следующую команду в терминале для кодирования ключа API в формат Base64:

```bash
echo -n "<API_KEY>:" | base64
```

Замените `<API_KEY>` на ваш фактический ключ API.

### Запрос данных через API

**Шаблон API-запроса:**
```bash
curl -X GET 'https://<URL_сайта>/api/<endpoint>' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

**Объяснение параметров:**
- `<URL_сайта>` — адрес сайта, например, `e-cat.co.il`.
- `<endpoint>` — конечная точка API (например, `products`, `customers`).
- `<API_KEY>` — ключ API, закодированный в формате Base64.

### Пример вызова API
Для получения списка продуктов на сайте `e-cat.co.il`:
```bash
curl -X GET 'https://e-cat.co.il/api/products' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

### Пример кода на Python

```python
import requests
import base64

# Параметры запроса
url_сайта = 'https://e-cat.co.il'
endpoint = 'products'
api_key = '<API_KEY>'

# Кодирование ключа API в Base64
api_key_encoded = base64.b64encode(f'{api_key}:'.encode()).decode()

# Формирование заголовка Authorization
headers = {
    'Authorization': f'Basic {api_key_encoded}'
}

# Выполнение запроса
response = requests.get(f'{url_сайта}/api/{endpoint}', headers=headers)

# Обработка ответа
if response.status_code == 200:
    print(response.json())
else:
    print(f'Ошибка: {response.status_code}')
    print(response.text)
```

## Рекомендации по безопасности

- Никогда не передавайте файл `credentials.kdbx` третьим лицам. ❗
- Убедитесь, что файл находится в защищенном месте, доступном только вам. (папка `secrets` в корне проекта исключена из `git`)
- Регулярно обновляйте ключи API и пароли для базы данных.
- Настройте права доступа к папке `secrets` так, чтобы только авторизованные пользователи могли получить к ней доступ.

## Обновление ключей API и паролей

Рекомендуется обновлять ключи API и пароли для базы данных не реже одного раза в три месяца, а также при возникновении подозрений на компрометацию.

## Дополнительно

Если у вас возникли вопросы или трудности с подключением, ознакомьтесь с [официальной документацией PrestaShop API](https://devdocs.prestashop.com/), где представлена информация о доступных конечных точках и способах работы с ними.

Документация актуальна для PrestaShop версии 1.7 и выше.

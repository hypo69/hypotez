## Описание процесса обработки данных

### Общее описание
Процесс предназначен для обработки запросов на получение данных о продуктах и их переводов на русский (`ru`) и иврит (`he`).
Он включает в себя взаимодействие с внешней моделью (например, Google Gemini), проверку валидности полученных данных и логирование ошибок.

### Этапы процесса:

1.  **Получение запроса (A)**:
    -   Начало процесса с получения запроса на обработку списка продуктов (`products_list`).

2.  **Обработка запроса (B)**:
    -   Запрос передается на обработку модели. Модель должна вернуть данные, содержащие переводы продуктов на `ru` и `he`.

3.  **Получение ответа (C)**:
    -   Ожидание ответа от модели.

4.  **Проверка на отсутствие ответа (C -> D)**:
    -   Если ответ от модели не получен, фиксируется ошибка `no response from gemini`.
    -   Производится повторный запрос к модели (E). Количество попыток ограничено (`attempts - 1`).

5.  **Проверка на валидность данных (C -> F)**:
    -   Если данные, полученные от модели, невалидны, фиксируется ошибка `Error in data from gemini`.
    -   Производится повторный запрос к модели (E).

6.  **Анализ структуры данных (G, L)**:
    -   Проверка формата полученных данных. Данные могут быть в виде списка или объекта.

7.  **Обработка данных в виде списка (G -> H)**:
    -   Если данные представлены в виде списка, проверяется, содержит ли список два элемента (`ru` и `he`).
    -   Если список содержит два элемента, они извлекаются (I).
    -   Если структура списка невалидна, фиксируется ошибка `Проблема парсинга ответа` (K).

8.  **Обработка данных в виде объекта (L -> M)**:
    -   Если данные представлены в виде объекта, извлекаются значения для ключей `ru` и `he`.
    -   Если данные в объекте невалидны, фиксируется ошибка `Invalid ru or he data` (N).

9.  **Возврат результата (O)**:
    -   Возврат извлеченных данных `ru` и `he`.

### Используемые компоненты:

-   `src.logger`: для логирования ошибок и информации о процессе.
-   Внешняя модель (например, Google Gemini): для получения переводов продуктов.

### Обработка ошибок:

-   В случае отсутствия ответа от модели (`no response from gemini`), фиксируется ошибка с использованием `logger.error`.
-   В случае невалидных данных (`Error in data from gemini`), фиксируется ошибка с использованием `logger.error`.
-   В случае проблем с парсингом ответа (`Проблема парсинга ответа`), фиксируется ошибка с использованием `logger.error`.
-   В случае невалидных данных `ru` или `he` (`Invalid ru or he data`), фиксируется ошибка с использованием `logger.error`.

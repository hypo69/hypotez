# Модуль для создания файла SUMMARY.md для mdBook
## Обзор

Модуль предназначен для автоматического создания или обновления файла `SUMMARY.md`, который используется в mdBook для организации структуры книги. Он рекурсивно обходит указанную директорию, находит все файлы с расширением `.md` и формирует на их основе структуру глав и подглав в файле `SUMMARY.md`.

## Подробнее

Этот модуль упрощает процесс создания и обновления структуры книги в формате mdBook. Он автоматически генерирует файл `SUMMARY.md` на основе структуры каталогов и имен файлов `.md`, что позволяет избежать ручного редактирования этого файла при изменении структуры книги. Модуль преобразует имена файлов в удобочитаемые заголовки глав, заменяя символы подчеркивания на пробелы и делая первую букву каждого слова заглавной.

## Функции

### `make_summary`

**Назначение**: Рекурсивно обходит папку с `.md` файлами и создает файл `SUMMARY.md` с главами на основе этих файлов. Если файл `SUMMARY.md` уже существует, он будет перезаписан.

**Параметры**:
- `src_dir` (Path): Путь к папке с исходными `.md` файлами. По умолчанию используется значение `src_path` определенное в модуле.
- `summary_file` (Path): Путь для сохранения файла `SUMMARY.md`. По умолчанию используется значение `summary_path` определенное в модуле.

**Возвращает**:
- `None`: Функция ничего не возвращает.

**Как работает функция**:

1.  **Проверка существования файла**: Проверяет, существует ли файл `SUMMARY.md` по указанному пути. Если файл не существует, он будет создан. Если файл существует, он будет перезаписан.
2.  **Открытие файла для записи**: Открывает файл `SUMMARY.md` в режиме записи с кодировкой UTF-8.
3.  **Запись заголовка**: Записывает заголовок "# Summary\n\n" в файл `SUMMARY.md`.
4.  **Рекурсивный обход файлов**: Рекурсивно обходит все файлы с расширением `.md` в указанной директории.
5.  **Исключение SUMMARY.md**: Пропускает сам файл `SUMMARY.md`, чтобы избежать рекурсивной ссылки.
6.  **Формирование пути и названия главы**: Формирует относительный путь к файлу и преобразует имя файла в удобочитаемый формат для названия главы.
7.  **Определение уровня вложенности**: Определяет уровень вложенности главы на основе количества поддиректорий в относительном пути.
8.  **Запись главы в файл**: Записывает строку с информацией о главе в файл `SUMMARY.md`, используя относительный путь и отступы для правильного отображения в структуре книги.
9.  **Завершение**: Выводит сообщение об успешном создании или перезаписи файла `SUMMARY.md`.

**Примеры**:

```python
from pathlib import Path
from src import gs
from hypotez.toolbox.mdbook_create_summary import make_summary

# Путь к директории с файлами .md
src_path = Path(gs.path.root / 'docs' / 'gemini' / 'consultant' / 'ru' / 'src')

# Путь для сохранения SUMMARY.md
summary_path = src_path / 'SUMMARY.md'

# Пример использования с пользовательскими путями
custom_src_dir = Path('/path/to/your/md/files')
custom_summary_file = Path('/path/to/your/SUMMARY.md')
make_summary(custom_src_dir, custom_summary_file)

# Пример использования с путями по умолчанию
make_summary()
```
## Параметры
- **src_path** (Path): Путь к директории с файлами `.md`. Используется для определения местоположения файлов, на основе которых будет генерироваться `SUMMARY.md`.
- **summary_path** (Path): Путь к файлу `SUMMARY.md`. Определяет, где будет создан или перезаписан файл `SUMMARY.md`.

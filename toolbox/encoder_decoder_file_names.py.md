# Модуль для кодирования и декодирования путей файлов

## Обзор

Модуль предоставляет класс `PathEncoderDecoder`, который используется для кодирования и декодирования путей файлов. Он применяет хэширование для создания коротких идентификаторов путей и хранит соответствия между идентификаторами и путями в базе данных SQLite. Это полезно для сокращения длины путей и упрощения работы с файлами в системе.

## Подробнее

Класс `PathEncoderDecoder` обеспечивает функциональность для кодирования путей файлов в короткие идентификаторы и обратно. Он использует базу данных SQLite для хранения соответствий между исходными путями и их закодированными представлениями. Это позволяет эффективно управлять и восстанавливать пути файлов, особенно когда необходимо работать с большими объемами данных или когда пути должны быть сокращены для удобства.

## Классы

### `PathEncoderDecoder`

**Описание**: Класс для кодирования и декодирования путей с использованием хэширования и хранения данных в SQLite базе данных.

**Атрибуты**:
- `db_path` (str): Путь к SQLite базе данных для хранения таблицы соответствий.

**Методы**:
- `__init__(db_path: str = 'path_mapping.db')`: Инициализация класса и базы данных.
- `_initialize_database()`: Инициализация базы данных, создание таблицы, если она отсутствует.
- `_save_to_database(short_id: str, file_path: str)`: Сохраняет хэш и путь в базу данных.
- `_fetch_from_database(short_id: str) -> Optional[str]`: Извлекает путь из базы данных по короткому идентификатору.
- `encode(file_path: str) -> str`: Кодирует путь в короткий идентификатор.
- `decode(short_id: str) -> Optional[str]`: Декодирует короткий идентификатор обратно в путь.
- `clear_mapping()`: Очищает таблицу соответствий.

**Принцип работы**:

Класс `PathEncoderDecoder` инициализируется с путем к базе данных SQLite, где будет храниться таблица соответствий между полными путями файлов и их короткими идентификаторами. При инициализации создается база данных, если она еще не существует, и таблица `path_mapping` с полями `id` (TEXT PRIMARY KEY) и `file_path` (TEXT NOT NULL).

Метод `encode` принимает полный путь к файлу, генерирует MD5 хэш этого пути, берет первые 8 символов хэша и создает короткий идентификатор в формате `id-{hash}`. Затем этот идентификатор и исходный путь сохраняются в базе данных.

Метод `decode` принимает короткий идентификатор и извлекает соответствующий полный путь из базы данных.

Метод `clear_mapping` очищает таблицу соответствий, удаляя все записи.

- Возвращает короткий идентификатор.

## Примеры

```python
# Создание экземпляра класса
encoder_decoder = PathEncoderDecoder()

# Кодирование пути
original_path = 'src/same_folder/same_sub_folder/same-file.ext'
encoded = encoder_decoder.encode(original_path)
print(f'Кодированный путь: {encoded}')

# Декодирование пути
decoded = encoder_decoder.decode(encoded)
print(f'Декодированный путь: {decoded}')

# Модуль для создания индексных файлов RST

## Обзор

Модуль `rst_indexer.py` предназначен для автоматического создания индексного файла `index.rst` в формате reStructuredText (RST), который используется в документации Sphinx. Он рекурсивно обходит указанный каталог, находит все файлы Python и добавляет их в `index.rst` для создания структуры `toctree`.

## Подробнее

Этот модуль упрощает процесс создания документации для проектов на Python, автоматически генерируя оглавление с перечислением всех модулей. Это позволяет избежать ручного обновления списка модулей в документации при добавлении новых файлов.

## Функции

### `create_index_rst`

```python
def create_index_rst(start_dir: str) -> None:
    """
    Recursively traverses all subdirectories from the start directory, reads all *.py files,
    and creates an index.rst file in the `docs` directory that lists all these files
    using the Sphinx `toctree` structure. Logs the process throughout.

    Args:
        start_dir (str): The root directory to start the traversal from.

    Returns:
        None

    Example:
        >>> create_index_rst(os.getcwd())
    """
    ...
```

**Назначение**: Рекурсивно обходит все подкаталоги от начального каталога, считывает все \*.py файлы и создает файл index.rst в каталоге `docs`, который перечисляет все эти файлы, используя структуру `toctree` Sphinx. Логирует процесс на протяжении всей работы.

**Параметры**:
- `start_dir` (str): Корневой каталог, с которого начинается обход.

**Возвращает**:
- `None`

**Вызывает исключения**:
- `Exception`: Если возникает ошибка при создании файла `index.rst`.

**Как работает функция**:

1.  **Определение путей**: Функция получает начальный путь (`start_dir`), создает пути к каталогу `docs` и файлу `index.rst`.
2.  **Создание каталога `docs`**: Если каталог `docs` не существует, он создается.
3.  **Запись в `index.rst`**: Открывается файл `index.rst` для записи, и в него записывается заголовок в формате Sphinx, необходимый для создания оглавления.
4.  **Обход каталогов**: Функция рекурсивно обходит все подкаталоги начального каталога в поисках файлов Python.
5.  **Фильтрация файлов**: Из найденных файлов выбираются только те, которые имеют расширение `.py` и не содержат символы `(` и `)` в имени (чтобы исключить некоторые типы файлов).
6.  **Формирование путей модулей**: Для каждого найденного файла вычисляется относительный путь, который используется для формирования имени модуля в формате, требуемом Sphinx.
7.  **Добавление модулей в `index.rst`**: Имена модулей добавляются в файл `index.rst` с использованием синтаксиса `toctree` Sphinx.
8.  **Обработка ошибок**: В случае возникновения исключения в процессе создания файла, оно логируется.

**Примеры**:

```python
import os
create_index_rst(os.getcwd())
```

В этом примере функция `create_index_rst` вызывается с текущим рабочим каталогом в качестве начального каталога. Это приведет к созданию файла `index.rst` в каталоге `docs`, который будет содержать список всех файлов Python в текущем рабочем каталоге и его подкаталогах.

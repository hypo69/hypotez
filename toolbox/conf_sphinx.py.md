# Модуль конфигурации Sphinx

## Обзор

Этот модуль содержит конфигурацию для генерации документации Sphinx. Он определяет параметры проекта, такие как имя, авторские права, автор и версия релиза, а также общие настройки Sphinx, такие как используемые расширения, пути к шаблонам и исключаемые шаблоны. Кроме того, модуль предоставляет функцию для игнорирования файлов с определенными шаблонами при генерации документации.

## Подробней

Модуль предназначен для настройки процесса генерации документации с использованием Sphinx. Он загружает параметры проекта из файла `settings.json` и определяет различные параметры конфигурации, такие как тема оформления, пути к статическим файлам и расширения Sphinx. Кроме того, модуль предоставляет функцию `skip_files` для исключения определенных файлов из документации на основе их имени.

## Функции

### `skip_files`

```python
def skip_files(app, what, name, obj, skip, options):
    """
    Функция-обработчик события 'autodoc-skip-member'.
    Используется для игнорирования членов документации, соответствующих определённым шаблонам.

    Args:
        app: Объект приложения Sphinx
        what: Тип документации (например, 'module', 'class', 'exception', 'function', 'method', 'attribute')
        name: Имя члена документации
        obj: Объект члена документации
        skip: Логическое значение, указывающее, следует ли пропустить этот член по умолчанию
        options: Опции автодокументации

    Returns:
        bool: True, если член должен быть пропущен; иначе возвращает значение `skip`
    """
    ...
```

**Назначение**:
Эта функция определяет, следует ли пропускать определенный член документации (например, функцию или класс) при генерации документации Sphinx.

**Параметры**:
- `app`: Объект приложения Sphinx.
- `what`: Тип документации (например, `'module'`, `'class'`, `'exception'`, `'function'`, `'method'`, `'attribute'`).
- `name`: Имя члена документации.
- `obj`: Объект члена документации.
- `skip`: Логическое значение, указывающее, следует ли пропустить этот член по умолчанию.
- `options`: Опции автодокументации.

**Возвращает**:
- `bool`: `True`, если член должен быть пропущен; иначе возвращает значение `skip`.

**Как работает функция**:
Функция проверяет, содержит ли имя члена круглые скобки или соответствует ли оно другим шаблонам. Если имя соответствует одному из этих шаблонов, функция возвращает `True`, указывая Sphinx пропустить этот член документации. В противном случае функция возвращает исходное значение `skip`, предложенное Sphinx.

**Примеры**:
```python
# Пример использования функции skip_files
def skip_files(app, what, name, obj, skip, options):
    if fnmatch.fnmatch(name, '*(*.*)') or fnmatch.fnmatch(name, '*(*).*'):
        return True
    return skip
```

### `setup`

```python
def setup(app):
    """
    Функция настройки, которая подключает функцию `skip_files` к событию `autodoc-skip-member`.

    Args:
        app: Объект приложения Sphinx
    """
    ...
```

**Назначение**:
Эта функция является точкой входа для Sphinx и используется для подключения функции `skip_files` к событию `autodoc-skip-member`.

**Параметры**:
- `app`: Объект приложения Sphinx.

**Как работает функция**:
Функция использует метод `connect` объекта приложения Sphinx для подключения функции `skip_files` к событию `autodoc-skip-member`. Это гарантирует, что функция `skip_files` будет вызываться для каждого члена документации, чтобы определить, следует ли его пропустить.

**Примеры**:
```python
# Пример использования функции setup
def setup(app):
    app.connect('autodoc-skip-member', skip_files)
```

## Переменные

- `project` (str): Название проекта. Загружается из `settings.json` или устанавливается по умолчанию как "hypotez".
- `copyright` (str): Авторские права. Загружаются из `settings.json` или устанавливаются по умолчанию как "2024, hypo69".
- `author` (str): Автор проекта. Загружается из `settings.json` или устанавливается по умолчанию как "2024, hypo69".
- `release` (str): Версия релиза проекта. Загружается из `settings.json` или устанавливается по умолчанию как "2024, hypo69".
- `extensions` (list): Список расширений Sphinx, используемых для генерации документации. В данном случае используется `sphinx.ext.autodoc` для автогенерации документации из docstring'ов.
- `templates_path` (list): Путь к директории с шаблонами Sphinx.
- `exclude_patterns` (list): Список шаблонов файлов и папок, которые следует исключить из процесса генерации документации.
- `available_themes` (dict): Словарь доступных тем оформления Sphinx.
- `html_theme` (str): Тема оформления Sphinx, используемая для генерации документации.
- `html_static_path` (list): Путь к директории со статическими файлами (CSS, изображения и т.д.).

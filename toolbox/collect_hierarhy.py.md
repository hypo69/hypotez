# Модуль collect_hierarhy

## Обзор

Модуль `collect_hierarhy` предназначен для рекурсивного сбора иерархии файлов из директории `src/utils` и копирования этих файлов в директорию `prod`. В процессе сбора игнорируются определенные папки и файлы, соответствующие заданным критериям. Иерархия файлов сохраняется в формате JSON в файле `file_hierarchy.json` внутри папки `prod`.

## Подробней

Этот модуль используется для создания структуры проекта, необходимой для работы с утилитами и скриптами. Он позволяет собрать все необходимые файлы (с расширениями `.py`, `.json`, `.md`, `.dot`, `.mer`) из директории `src/utils`, исключая при этом временные и неиспользуемые файлы (например, папки `profiles`, `__pycache__`, `_experiments` и файлы, начинающиеся с `___`). Копирование файлов сохраняет исходную структуру директорий, что упрощает развертывание и поддержку проекта.

## Функции

### `collect_and_copy_files`

```python
def collect_and_copy_files(directory: Path, target_directory: Path) -> dict:
    """Рекурсивно проходит по директории, собирает иерархию файлов и копирует их в целевую директорию.

    Args:
        directory (Path): Путь к директории, которую необходимо обработать.
        target_directory (Path): Путь к целевой директории, куда будут скопированы файлы.

    Returns:
        dict: Словарь, представляющий иерархию файлов и папок. Ключами являются имена файлов и папок, значениями - вложенные словари для папок и `None` для файлов.
    """
    ...
```

**Назначение**: Рекурсивно обходит указанную директорию, собирает иерархию файлов и копирует файлы с определенными расширениями в целевую директорию.

**Параметры**:
- `directory` (Path): Путь к директории, которую необходимо обработать.
- `target_directory` (Path): Путь к целевой директории, куда будут скопированы файлы.

**Возвращает**:
- `dict`: Словарь, представляющий иерархию файлов и папок. Ключами являются имена файлов и папок, значениями - вложенные словари для папок и `None` для файлов.

**Как работает функция**:

1.  Инициализирует пустой словарь `hierarchy` для хранения иерархии файлов и папок.
2.  Перебирает все элементы (файлы и поддиректории) в указанной директории.
3.  Если элемент является директорией, проверяет, не входит ли его имя в список исключений (`profiles`, `__pycache__`, `_experiments`) и не начинается ли с `___`, а также не содержит ли символ `*`. Если все проверки пройдены, рекурсивно вызывает `collect_and_copy_files` для обработки содержимого этой директории и добавляет результат в словарь `hierarchy`.
4.  Если элемент является файлом, проверяет, имеет ли он одно из разрешенных расширений (`.py`, `.json`, `.md`, `.dot`, `.mer`), не начинается ли его имя с `___`, и не содержит ли символы `*`, `(` или `)`. Если все проверки пройдены, добавляет имя файла в словарь `hierarchy` со значением `None`, создает необходимые поддиректории в целевой директории и копирует файл в целевую директорию.
5.  Возвращает словарь `hierarchy`, представляющий иерархию файлов и папок.

**Примеры**:

```python
from pathlib import Path
# Пример вызова функции
src_dir = Path('/путь/к/src')
target_dir = Path('/путь/к/target')
hierarchy = collect_and_copy_files(src_dir, target_dir)
print(hierarchy)
```

### `main`

```python
def main():
    """Выполняет сбор и копирование файлов из src/utils в prod и сохраняет иерархию в file_hierarchy.json."""
    ...
```

**Назначение**: Выполняет основные действия по сбору и копированию файлов из директории `src/utils` в директорию `prod`, а также сохраняет иерархию файлов в файл `file_hierarchy.json`.

**Параметры**:
- Функция не принимает параметров.

**Возвращает**:
- Функция ничего не возвращает.

**Как работает функция**:

1.  Определяет исходную директорию `src_directory` как `src/utils` внутри корня проекта.
2.  Определяет директорию для сохранения результатов `project_structure_directory` как `prod` внутри `src_directory`.
3.  Вызывает функцию `collect_and_copy_files` для сбора иерархии файлов и копирования их в `project_structure_directory`.
4.  Определяет путь для сохранения JSON-файла с иерархией `json_output_path` как `file_hierarchy.json` внутри `project_structure_directory`.
5.  Сохраняет полученную иерархию файлов в формате JSON в файл `file_hierarchy.json` с использованием функции `j_dumps`.

**Примеры**:

```python
# Пример вызова функции
main()
```

## Переменные

- `MODE` (str): Указывает режим работы. В данном случае установлен в `development`.
- `src_directory` (Path): Путь к исходной директории `src/utils`, из которой собираются файлы.
- `project_structure_directory` (Path): Путь к директории `prod`, в которую копируются файлы и сохраняется иерархия.
- `file_hierarchy` (dict): Словарь, содержащий иерархию файлов и папок, собранную функцией `collect_and_copy_files`.
- `json_output_path` (Path): Путь к файлу `file_hierarchy.json`, в который сохраняется иерархия файлов в формате JSON.
